# 安全政策

## 支持范围

当前支持主干分支和最近公开发布版本的安全问题处理。历史实验分支、个人修改版本和未授权商业版本不在默认支持范围内。

## 私下报告

如发现以下问题，请通过私下渠道报告，不要公开提交可复现攻击细节：

1. 认证绕过、越权、租户数据泄露。
2. API Key、Token、密码、私钥、数据库连接信息泄露。
3. SQL 注入、命令注入、XSS、SSRF、任意文件读写。
4. OAuth2/OIDC、签名验签、限流、防重放相关缺陷。
5. 可导致生产环境拒绝服务、数据破坏或权限提升的问题。

报告方式：

- 邮箱：cloudkj@163.com
- 标题建议：`[ZHYC Security] 漏洞摘要`

## 报告内容

请尽量提供：

1. 受影响版本、模块和接口。
2. 复现步骤或最小复现请求。
3. 影响范围和潜在风险。
4. 你已经采取的临时缓解措施。

## 处理承诺

收到报告后，我们会尽快确认问题、评估影响、安排修复，并在合适时机公开安全公告。请在修复发布前避免公开漏洞细节。
